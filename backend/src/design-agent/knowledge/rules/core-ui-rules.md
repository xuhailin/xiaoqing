# XiaoQing Design Skill v1 - Core UI Rules

本文件定义“小晴 UI 不可自由发挥的硬约束”。默认所有页面先遵守这里，再叠加 preset 偏移。

## 1. 页面层级总规则

稳定层级必须保持为：

1. 页面背景
2. 页面 header
3. 一级内容区
4. 内容内的 list / message / detail
5. 仅用于强调的信息块

禁止出现：

- header 比主内容更像“卡片主体”
- 一级内容区和二级内容区同时抢主视觉
- 一个页面同时存在两套不同的 panel 语言

### 面板语义命名（全局统一词汇）

从现在起统一使用三层面板语义，不再自由命名：

- `no-panel`：无容器层（透明、无边框、无阴影）
- `sub-panel`：次级容器层（用于主区域左侧区域、筛选区、轻承载区）
- `panel`：标准内容卡片层（列表项、详情块、可交互对象）

约束：

- `main-left-section`（主区域左侧区域）默认必须使用 `sub-panel`
- `main-left-section` 内的列表项默认使用 `panel`
- 禁止 `main-left-section` 再套额外同级 `sub-panel`，避免双层壳

## 2. Header 与内容区关系

header 的职责只有三个：

- 说明当前页面是什么
- 暴露一级筛选或主操作
- 提供上下文，不承载大段内容

硬约束：

- header 必须比内容区更轻，不能比主 panel 更重
- header 允许使用弱背景或轻磨砂，但不允许使用厚阴影和强边框
- 页面主标题只出现一次
- header 到主内容的过渡靠 spacing，不靠多层边框或装饰条
- 一级内容区必须是页面真正的视觉主体

推荐间距：

- header 内部：`12 / 16`
- header 到首个 section：`16 / 24`
- section 到 section：`16 / 24 / 32`

## 3. 卡片使用边界

卡片只允许用于以下场景：

- 单条对象需要被点击、选中、比较
- 状态块需要与主背景明确分离
- 工具结果、工作项、摘要块需要单独承载
- 详情内容需要被 panel 化承接

禁止场景：

- 为了“完整感”给整个页面再包一层大卡片
- chat 主流区域外面再套 panel
- header 外再加独立装饰卡片
- 在一个卡片内部再重复铺满小卡片，形成卡片套卡片

数量约束：

- 同一视口内，强视觉卡片不超过 2 层
- 同一组列表项只允许一种卡片样式
- 非状态型页面中，大面积纯装饰卡片为 0
- 若区域为 `main-left-section`，容器层必须是 `sub-panel`，条目层必须是 `panel`

## 4. 渐变使用规则

渐变只能出现在以下区域：

- 页面最外层 shell 背景
- workbench header 的弱层级背景
- 选中的主按钮或主行动按钮
- 少量 hero / reflection / summary 型强调模块

渐变禁止出现在：

- 普通正文卡片
- 列表项默认态
- input、textarea、select 主体
- 大面积正文阅读区
- badge、tag、表格行、二级按钮

强度约束：

- 同一页面只允许 1 个主环境渐变
- 同一屏内最多 1 个强调渐变块
- 渐变只负责“气氛”和“层级”，不能替代信息分组

## 5. 阴影与边框规则

- 默认优先细边框，再考虑轻阴影
- 阴影只用于把容器从背景中抬起，不能制造“漂浮 UI”
- chat message 使用轻边框或轻背景即可，不默认依赖重阴影
- dark 模式中优先提高边界对比，不通过加重 glow 解决层级

禁止：

- 同时使用强阴影 + 强渐变 + 强描边
- hover 态通过大位移和大阴影制造夸张反馈
- 为每个局部模块单独发明阴影体系

## 6. 信息密度控制

小晴默认是“中高密度、秩序优先”。

硬约束：

- spacing 只允许使用 `4 / 8 / 12 / 16 / 24 / 32`
- radius 只允许使用 token 档位，不新发明 `10 / 14 / 18 / 22`
- 页面标题、区块标题、正文、元信息必须回到既有字号层级
- 一页里同类控件尺寸必须一致

密度原则：

- `chat`: 流式，内容密，但 chrome 轻
- `workbench`: 信息密且结构强，优先可扫读
- `memory`: 信息密度中等，阅读性优先于操作感

## 7. 颜色与主题约束

- 颜色必须来自 theme token 或语义 token，不在业务层直接造色
- light / dark 是同一体系的两种投影，不是两套独立风格
- dark 模式允许更强边界，但不允许整体变“赛博霓虹”
- 状态色只通过 `primary / success / warning / error / neutral` 语义表达


## 8. 动效与反馈

- 默认使用短时、低位移、低注意力的反馈
- 页面级动画只用于加载进入或轻量 section reveal
- hover / active 的存在感小于信息本身

禁止：

- 弹跳式交互
- 大范围 blur/glow 扩散
- 为了“高级感”给所有卡片加浮起动效

## 9. 最小修改原则

任何设计修改都必须先回答：

1. 能否只通过 token、共享 primitive、已有布局关系解决
2. 能否只改一个层级，不牵动整页
3. 能否通过删除视觉噪音而不是新增装饰来提升完成度

如果以上任一答案为“能”，就不应继续做更大改动。
