# XiaoQing Design Skill v1 - Page Type Patterns

本文件定义不同页面类型的风格差异。它们共享同一套 token 和规则，只在层级、密度与强调方式上不同。

## 1. Chat 页面

### 目标气质

- 轻、流动、陪伴感
- 内容优先，不做工作台大壳子
- 让消息、状态块、输入区自然分层

### 结构规则

- chat 主流区域保持透明或极轻背景
- message 是基础层，状态块和 work item 才能升级成卡片
- composer 是工具区，不是页面主视觉
- 左侧会话区域若作为主区域左侧区域，命名为 `main-left-section`，容器使用 `sub-panel`，会话项使用 `panel`

### 卡片规则

- 禁止“外层大卡片 + 内层消息卡片”
- 普通 message 不做厚重卡片
- 允许对 `memory / world-state / reflection / work-item` 这类高价值块做轻 panel

### 渐变规则

- 允许页面背景有非常弱的环境渐变
- 允许 reflection / summary 块使用一次强调渐变
- 不允许把 message 列表做成渐变舞台

### 密度规则

- message 间距以 `8 / 12` 为主
- composer 与消息区间距小于 workbench 页面 section 间距
- 控件数量应克制，默认一屏只保留一个主输入焦点

## 2. Workbench 页面

### 目标气质

- 稳定、专业、可扫读
- 结构强于情绪表达
- 适合长时间操作和多模块并列

### 结构规则

- header 可以比 chat 稍强，但仍不能压过主内容
- 一级区块允许使用 `workbench surface`
- 列表区、详情区、状态区可以并排，但必须共享同一套 panel 语言
- 若存在主区域左侧区域，统一使用 `main-left-section + sub-panel`，列表项统一 `panel`

### 卡片规则

- 允许 panel 承载一级功能区
- 列表项与详情容器可以同时存在，但样式要有明确主次
- 不允许每个小功能都独立造一套“微卡片”

### 渐变规则

- 允许 shell 背景和 header 使用弱渐变
- 一级主 panel 默认不用强渐变，优先细边框 + 轻阴影
- 强调渐变只保留给主 CTA 或关键摘要块

### 密度规则

- 默认使用 `compact` 或 `balanced` 密度
- 过滤器、标签、状态、表单控件的高度与间距必须统一
- 同屏可并列更多信息，但阅读区宽度不要过度拉满

## 3. Memory 页面

### 目标气质

- 安静、个人化、内容导向
- 比 workbench 更柔和，但不松散
- 允许轻微温度感，不允许“治愈系堆装饰”

### 结构规则

- header 应明显轻于正文内容
- 列表、详情、关系内容应更接近阅读器，而不是控制台
- 重点放在内容块的呼吸感和阅读顺序
- memory 设置型页面左侧导航区域统一视为 `main-left-section`，应用 `sub-panel`；导航项使用 `panel`

### 卡片规则

- 允许列表项和详情块使用安静卡片
- 不鼓励把所有内容切成很多独立卡片
- 长内容优先用 section 分组，而不是无限新增容器

### 渐变规则

- 仅允许最外层背景有极弱渐变
- 详情区、正文区默认不用渐变
- 高亮只可用于少量 summary / relation 概览

### 密度规则

- 默认使用 `balanced` 或 `relaxed`
- 文本行长、段间距、标题留白比 chat/workbench 更宽松
- 操作按钮要退后，不能和正文争抢注意力

## 4. 页面类型判定优先级

当一个页面同时包含多种属性时，按以下优先级判定主风格：

1. 用户在这里主要是“聊天”还是“操作”
2. 页面主要是“工作流控制”还是“内容阅读”
3. 页面最长停留对象是“消息流”“任务面板”还是“长期内容”

映射建议：

- 以消息流为核心：`chat`
- 以任务、配置、状态操作为核心：`workbench`
- 以记忆、关系、内容浏览为核心：`memory`
