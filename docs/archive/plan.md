# 小晴下周 Sprint 计划：认知链路稳定化

## Sprint Goal
本周目标不是扩展新能力，而是完成小晴 chat 主链路、质量监控、devagent 配置、私人 skill 调研的稳定化建设，让系统进入“可控、可测、可维护”的状态。

---

## Phase 1：认知链路梳理与定型
### 目标
梳理 chat 从输入到输出的完整链路，明确 intent、reasoning、system self、tool decision、expression 各层职责与顺序。

### 输出
- 当前链路图
- 目标链路图
- 重构方案文档

### 验收
- 能清楚解释 thinking 为什么没有体现在输出里
- 能清楚说明 persona 在哪里注入
- 能形成统一标准链路

---

## Phase 2：质量监控与防退化测试
### 目标
建立从意图识别到输出回复的最小健康检查机制，防止系统改动后无感退化。

### 输出
- QA 测试样例文档
- 检测维度定义
- 手动/半自动复测流程
- 退化判定标准

### 验收
- 至少 8~12 条核心样例
- 覆盖 intent / reasoning / persona / tool / devagent
- 每次改动后可快速复测

---

## Phase 3：DevAgent 执行层收口
### 目标
为 devagent 增加明确的模型配置与默认执行规则，收敛 planner / executor / review 的职责边界。

### 输出
- 模型配置方案
- 默认执行策略
- assistant 与 devagent 边界说明
- 配置落地

### 验收
- 能明确每类任务默认模型
- executor 选择逻辑清晰
- 基础能力不再混入 devagent

---

## Phase 4：私人 skill 可行性调研
### 目标
验证打卡类私人 skill 的技术路径、风险与维护成本，为后续部署做判断依据。

### 输出
- 技术可行性分析
- 接入方案候选
- 风险评估
- 部署前提清单

### 验收
- 输出调研结论
- 明确是否进入下一阶段实现

---

## 本周不做
- 不扩展新能力
- 不推进企业微信正式接入
- 不做复杂平台化监控
- 不做激进 token 优化
- 不直接实现私人 skill 正式版
