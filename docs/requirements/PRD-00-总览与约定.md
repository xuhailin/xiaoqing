# PRD-00 总览与约定

**依赖**：无（后续文档均引用本文档）

---

## 一、项目目标

构建一个 **具备长期记忆能力的对话式 AI 系统**，用于**个人长期对话、思考记录与人格连续性维护**。

> 本项目 **不是** 通用 Agent 框架  
> 本项目 **不是** 自动化执行系统  
> 本项目 **不是** 多模型自治系统  

### 核心目标

1. **纯聊天**：自然语言对话，不参与写代码
2. 对话连续性（不是上下文堆叠）
3. 记忆可控（可查看、可编辑、可衰减）
4. token 使用长期可控
5. **人格可自动进化**，但受**初始约束池 + 进化池**约束，防止不良进化

### V1 能做什么（使用场景）

* **初始无关系、无依赖**：不预设用户画像（可为 0），用户可后续在记忆/画像中自行补充；伙伴通过聊天**慢慢陪伴**，逐步建立对用户的理解。
* **通过聊天进化人格**：在双池约束下，基于近期对话生成人格微调建议，经人工确认后写入人格层。
* **补充对你的理解、定义与汇总记忆**：手动（或规则）触发总结，从对话中抽取「未来仍有价值的判断」，写入阶段/长期记忆，可查看、可编辑、可回溯。
* 以上均在**本地**完成，数据存于本地 PostgreSQL；读物摄入（如看书学习）为后续阶段可选。

---

## 二、系统整体架构（逻辑）

```text
[ Angular Chat UI ]  纯聊天，无 code 模式
        ↓
[ Conversation Controller ]
        ↓
[ Prompt Router ]  →  chat / summary（仅后台）
        ↓
[ Memory Injection Layer ]
        ↓
[ LLM API ]
        ↓
[ Memory Summarizer（事件触发）]
        ↓
[ PostgreSQL ]
```

---

## 三、V1 明确不实现的内容

以下内容 **只列，不做**：

❌ 多 Agent 自主协商  
❌ Agent 自己决定写入长期记忆  
❌ embedding / 向量数据库  
❌ 自动情绪分析  
❌ **无约束的**人格进化（V1 只做「进化池约束下的自动进化」）  
❌ 定时全量总结（cron）  
❌ 多用户系统  
❌ 权限 / 登录  
❌ 云部署 / 多端同步  

---

## 四、非功能性要求

* **所有记忆必须可人工查看**
* **所有 prompt 有版本号**
* **任何总结都能回溯原始对话**
* **token 使用必须可预估**（上下文有明确 N 轮 + M 条记忆上限，请求前可估算并截断）
* **人格进化必须在进化池边界内**，且可查看 / 回滚

---

## 五、总指令（给 Cursor）

> 本项目是一个**纯聊天、带长期记忆**的人格伙伴，人格可自动进化但受**初始约束池 + 进化池**双重约束；优先保证可控性、可理解性与长期稳定性，避免无约束的自动化。
