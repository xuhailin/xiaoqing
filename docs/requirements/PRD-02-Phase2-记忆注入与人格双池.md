## 一、记忆注入策略

每一轮对话前，由**规则/配置**决定是否注入记忆（非模型自决）。建议策略（可配置）：

* 注入**最近 K 条 mid 记忆** + **全部 long 记忆**
* 按 **token 上限**截断：上下文 = 最近 N 轮对话 + 注入的记忆，总 token 不超过上限，请求前可估算（如 tiktoken 或字符粗算）并截断

## 二、人格系统（Persona）

### 人格是约束集合；人格可进化，但必须受控

##### 人格包含

* 行为边界（不能做什么）
* 语气风格
* 回应节奏
* 认知姿态（是否给结论）

##### 人格进化（双池防护）

* **可自动进化**：根据长期对话与记忆，在边界内微调人格表现。
* **初始约束池**：人格的起点约束，用户可选或配置（见下节）。
* **进化池**：规定**允许的进化方向**与**禁止的不良进化**，所有自动进化必须落在进化池内，防止人格跑偏或越界。

---

## 三、初始约束池 + 进化池（Persona 双池）

### 初始约束池（人格起点，可多选）

用于设定伙伴的**初始行为边界与倾向**，后续进化不得脱离进化池。

#### 人格约束（可多选）

* 不急于给建议
* 不使用“你应该”
* 不制造确定性幻觉
* 不总结用户的人生
* 不把情绪当问题修复
* 不扮演权威或导师
* 不强行正能量
* 不输出模板化鸡汤

#### 人格倾向（可选）

* 偏理性，但承认不确定性
* 更像“思考伙伴”而非“解答器”
* 优先追问澄清，而不是下结论
* 可指出逻辑矛盾，但不否定情绪

### 进化池（防止不良进化）

* **允许的进化方向**（示例）：在保持「思考伙伴」的前提下，可随用户偏好微调追问深度、理性/共情比例等。
* **禁止的进化**（示例）：不得变为说教型、不得增加「你应该」类表达、不得削弱「承认不确定性」、不得自动写入未经用户确认的长期价值观条目。
* 实现上：进化建议或自动微调需与进化池规则校验，通过后才生效；必要时可做**人工确认**再写入人格层。

### 人格进化机制（V1）

#### 触发方式

* 进化建议的触发来源支持：
  * 手动触发：用户在前端主动发起「根据近期对话更新人格」；
  * 总结后自动触发：无论手动总结还是自动总结完成后，系统会基于近期对话生成进化建议；
  * 记忆密度日检（可选）：当启用 `FEATURE_EVOLUTION_SCHEDULER` 时，会按密度生成待确认进化建议。
* 落地规则保持可控：用户偏好类字段（如 `preferredVoiceStyle` / `praisePreference` / `responseRhythm`）在建议生成后可直接应用；核心人格字段进入待确认队列，前端展示并经用户确认后再写入人格层。

#### 从对话到「进化建议」

* 输入：**近期对话片段**（如最近 N 轮或最近一次总结所涉对话）+ **当前人格描述**（来自人格层）+ **进化池**（允许方向 + 禁止项）。
* 方式：用 **LLM 一次调用**，prompt 要求其输出**一条人格微调建议**（自然语言，如「在保持思考伙伴的前提下，稍增加共情表达」），且不得违反进化池中的禁止项。
* 可选：可同时喂入长期记忆中与风格/偏好相关的条目，使建议更贴合「一直记得你」的上下文。

#### 校验与写入

* 进化建议生成后，先与**进化池规则校验**（规则或二次 LLM 校验均可）：若违反禁止项或超出允许方向，则丢弃或打回，不写入。
* 通过校验后：
  * 若变化落在用户偏好类字段，可直接写入用户画像偏好（无需用户确认）；
  * 若变化落在核心人格类字段，进入待确认队列并展示，用户确认后再写入人格层。
* 人格层存储形式：**可编辑的文本**（如 system prompt 中的人格段落，或 DB 中的 persona 文本字段）；进化 = 按建议更新该文本并写回。若后续改为结构化配置（如追问深度 0.7），则进化 = 更新配置项且在进化池允许范围内。

#### 与记忆 / 总结的联动

* 与「记忆总结」联动：在手动总结或自动总结完成并写入阶段/长期记忆后，系统会生成进化建议；用户偏好类变化可直接应用，核心人格变化会进入待确认建议，减少用户单独触发更新人格的次数。

## 四、前端（Angular）Phase2 要求

### 页面

* 单一聊天窗口
* 左侧（或抽屉）：
  * **当前人格摘要**
  * **已注入的记忆条目**

### 操作

* 纯聊天（无模式切换）
* 手动触发总结
* 查看 / 编辑记忆
* **查看 / 配置人格双池**（初始约束池、进化池）
* **手动触发「根据近期对话更新人格」**：展示进化建议；若存在由自动总结/密度日检产生的待确认核心人格变化，前端应一并展示并由用户确认后再写入（用户偏好类变化可能已自动落地）

---

## 本需求达成目标

* 每次对话前可按配置注入 mid/long 记忆，回复具备「记得你」的连续性
* 人格具备初始约束池与进化池：用户偏好类变化可自动落地；核心人格类变化经进化池校验 + 人工确认后写入人格层
* 用户可在前端看到人格摘要、本次注入的记忆、并可配置双池与触发人格进化
